# Dandelion 蒲公英运维管理系统 — 项目规范

## 项目信息
- 基于 RuoYi-Vue 框架二次开发
- 技术栈: Spring Boot 3.3 + Java 17 + MyBatis-Plus + Redis + JWT + MySQL 8.0
- 多模块 Maven 项目
- 使用 Flyway 管理数据库迁移
- 构建: Maven | 包名: `com.ruoyi`
- 主分支: `dandelion` | 开发分支: `feature/*`

---

## 分支策略
- `dandelion` — 主分支，用于生产发布
- `feature/*` — 功能开发分支（如 `feature/CMDB`）
- 代码提交到 `feature/*` 分支，通过 PR 合并到 `dandelion`

## 本地开发启动
- 使用 `test,local` 两个 profile 启动
- `application-test.yml` — 测试环境配置（指向远程）
- `application-local.yml` — 本地覆盖配置（不提交到 git）
- VM options: `-XX:TieredStopAtLevel=1`
- Program arguments: `--spring.profiles.active=test,local`

### 3. Controller 规范
```java
@RestController
@RequestMapping("/operations/<entity>")  // 模块前缀 + 实体名（单数、小写）
public class XxxController extends BaseController {

    @Resource
    private XxxService service;

    // 分页查询
    @PreAuthorize("@ss.hasPermission('module:entity:query')")
    @GetMapping("/list")
    public TableDataInfo pageable(Page<Entity> pageable, Entity entity) {
        return pageable(service.pageable(pageable, entity));
    }

    // 查询详情
    @PreAuthorize("@ss.hasPermission('module:entity:query')")
    @GetMapping("/{id}")
    public AjaxResult getInfo(@PathVariable Long id) {
        return AjaxResult.success(service.byId(id));
    }

    // 新增
    @Log(title = "标题", businessType = BusinessType.INSERT)
    @PreAuthorize("@ss.hasPermission('module:entity:add')")
    @PostMapping
    public AjaxResult add(@RequestBody Entity entity) {
        return toAjax(service.save(entity));
    }

    // 修改
    @Log(title = "标题", businessType = BusinessType.UPDATE)
    @PreAuthorize("@ss.hasPermission('module:entity:edit')")
    @PutMapping
    public AjaxResult edit(@RequestBody Entity entity) {
        return toAjax(service.updateById(entity));
    }

    // 删除
    @Log(title = "标题", businessType = BusinessType.DELETE)
    @PreAuthorize("@ss.hasPermission('module:entity:remove')")
    @DeleteMapping("/{ids}")
    public AjaxResult remove(@PathVariable Long[] ids) {
        return toAjax(service.deleteByIds(ids));
    }
}
```

**规则:**
- 类注解: `@RestController` + `@RequestMapping`
- 所有 Controller 继承 `BaseController`
- 使用 `@Resource` 注入 Service（不使用 `@Autowired`）
- 写操作加 `@Log` 和 `@PreAuthorize`
- 返回类型: 分页用 `TableDataInfo`，其他用 `AjaxResult`
- 导出方法返回 `void`，直接写入 `HttpServletResponse`
- 文件下载返回 `ResponseEntity<ClassPathResource>`

**规则:**
- 接口继承 `IService<Entity>`
- 实现继承 `ServiceImpl<Mapper, Entity>` 并实现接口
- 加 `@Service` 和 `@Slf4j`
- 事务方法加 `@Transactional(rollbackFor = Exception.class)`
- 使用 `baseMapper` 调用 MyBatis-Plus 方法

### 5. 领域实体规范
```java
@Data
@EqualsAndHashCode(callSuper = true)
@TableName(autoResultMap = true)
public class Entity extends SmartModel {
    private Long teamId;
    private String requesterName;

    @TableField(exist = false)  // 非数据库字段
    private String tempField;

    @TableField(typeHandler = ListToVarcharTypeHandler.class)  // 列表转逗号分隔
    private List<String> tags;

    @JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss")
    private LocalDateTime completeTime;
}
```

**继承层次:**
- `BaseModel` → `SmartModel` → 业务实体
  - `BaseModel`: id, createBy, createTime, updateBy, updateTime
  - `SmartModel`: + delFlag
- 不含 delFlag 的实体可直接继承 `BaseModel`

**字段规则:**
- Java 小驼峰命名（`teamId`），数据库自动映射为蛇形命名
- 非持久化字段加 `@TableField(exist = false)`
- 列表字段使用 `typeHandler = ListToVarcharTypeHandler.class`
- 日期字段加 `@JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss")`
- Excel 导出相关使用 `@ExcelProperty`、`@ExcelIgnoreUnannotated`

### 6. Mapper 规范
```java
@Repository
public interface XxxMapper extends BaseMapper<Entity> {
    Page<Entity> pageable(Page<Entity> pageable, @Param("entity") Entity entity);
    Entity byId(@Param("id") Long id);
}
```

XML 映射文件在 `src/main/resources/mapper/` 目录下：
- `namespace` 匹配 Mapper 全限定名
- 使用 `<resultMap>` 定义列到字段映射
- 广泛使用 `<if test="">` 动态 SQL
- 表别名约定：主表用小写实体名

### 7. 方法命名规范
| 前缀 | HTTP | 用途 |
|------|------|------|
| `pageable` / `list` | GET | 分页列表 |
| `getInfo` / `byId` | GET | 单条详情 |
| `add` / `save` | POST | 创建 |
| `edit` / `update` | PUT | 修改 |
| `remove` / `delete` | DELETE | 删除（软删除） |
| `export` | POST | 导出 |
| `upload` / `import` | POST | 导入 |
| `changeStatus` | PUT | 切换状态 |
| `start` / `pause` / `complete` / `approve` / `submit` / `assign` | PUT | 流程操作 |

### 8. URL 命名
- 模块路径: 小写、单数、蛇形命名（如 `/operations/event`、`/archive/machine_room`）
- 业务方法: 驼峰命名（如 `/exportReport`、`/updateDescriptionAndAssist`）
- 子资源: `/{id}/{action}` 或 `/{id}/{sub_resource}`
- 复合操作: 连字符分隔（如 `/batch-assign`）

### 9. 异常处理
- 业务异常: `throw new GlobalException("描述")`
- 框架统一捕获，返回 `AjaxResult`（code + msg）

### 10. 权限标识格式
```
<模块>:<实体>:<动作>
```
例: `operations:event:query`、`system:user:add`、`archive:device:export`

### 11. 操作日志格式
```java
@Log(title = "中文业务标题", businessType = BusinessType.INSERT)
// BusinessType: INSERT / UPDATE / DELETE / EXPORT / IMPORT / OTHER
```

### 12. 数据库规范
- 表名: 蛇形命名，全小写（`sys_user`、`change_process`）
- 列名: 蛇形命名，全小写（`machine_room_id`、`requester_name`）
- 主键: `id` bigint AUTO_INCREMENT
- 公共字段: create_by, create_time, update_by, update_time
- 逻辑删除: del_flag tinyint（0=存在, 1=删除）
- 使用 Flyway 管理迁移: `V{YYYYMMDDHHmmss}__{description}.sql`
- 迁移 SQL 放在 `dandelion-admin/src/main/resources/db/migration/`

### 13. 枚举规范
```java

    LOW(1L, "低"),

```
