# 多平台电话告警方案调研与对比

> 更新日期：2026-05-20

## 一、调研背景

需要实现定时/事件驱动的自动电话告警，核心需求：

1. **自动拨打电话**（PSTN，非 VoIP）
2. **判断是否接听**（接听确认）
3. **获取通话时长**（接听秒数）
4. **无需复杂资质**（或资质门槛低）

## 三、各方案详细分析

### 方案 1：天翼云（CTyun）

**API**：`POST /v4/monitor/send-message`

**当前状态**：✅ 已对接，堡垒机生产运行中

**优势**：

- 零资质要求，开通即用
- AK/SK 签名，安全性好
- 联系人组机制，灵活管理通知人员
- 已完整对接 `notify.py`，代码成熟
- 电话播报内容动态传入，无需审核模板

**劣势**：

- **无接听确认**：`statusCode: 800` 仅表示入队成功，无法判断用户是否接听
- **无通话时长**：不知道接听了几秒
- **无法区分未接/拒接/关机**

**适合场景**：对电话是否接通不敏感的通知场景

**所需参数**：

| 参数                          | 说明             |
| ----------------------------- | ---------------- |
| `CTYUN_AK`                  | 天翼云 AccessKey |
| `CTYUN_SK`                  | 天翼云 SecretKey |
| `CTYUN_MONITOR_GROUP_NAMES` | 联系人组名称     |

**调用示例**：

```bash
curl -X POST https://hzyz-nightingale.shsets.com/api/v1/call/trigger \
  -H "Content-Type: application/json" \
  -d '{"message":"告警通知","group":"值班组"}'
```

### 方案 2：企业微信（WeChat Work）❌

**API**：

- 发起呼叫：`POST /cgi-bin/pstncc/call`
- 查询状态：`POST /cgi-bin/pstncc/getstates`

**当前状态**：❌ 不可用

**不可行原因**：

企业微信的 PSTN 电话呼叫接口（`pstncc/call`）属于**「紧急通知」功能**。根据企业微信官方规定，紧急通知能力仅对**医疗类企业**开放，需提供医疗机构执业许可证等资质材料进行审核，普通企业无法开通此功能。
![1779239503894](image/多平台电话告警方案调研与对比/1779239503894.png)


> 上图：企业微信管理后台「紧急通知」功能说明，明确仅限医疗行业使用。

**关键限制**：

- 自建应用虽然可以创建，但**无法获得紧急通知授权**
- 调用 `pstncc/call` 接口会返回 `errcode: 48001`（API 功能未授权）
- 无任何绕过方式，属于行业准入硬性限制

**结论**：普通运维/IT 团队无法使用企业微信实现电话告警，该方案不可行。

### 方案 3：阿里云 VMS

**API**：`GET /?Action=SingleCallByTts`

**当前状态**：⚠️ 代码已写完，等待 TTS 模板资质审核

**优势**：

- 接听确认：`VoiceReport` 回调推送 `status_code` + `duration`
- 通话时长：`duration` 字段（秒）
- 7 种终态状态码（接听/挂断/拒接/关机/空号等）
- 中间状态回调（振铃/接听/挂断），实时感知
- 公共模式无需购买号码

**劣势**：

- **资质门槛高**：企业实名认证 + TTS 模板审核（1~3 工作日）
- TTS 模板必须审核通过，不能动态传任意内容（需要 `${message}` 通用模板）
- HMAC-SHA1 签名较复杂
- VoiceReport 回调需要公网可达的 HTTP 端点

**适合场景**：需要完整接听状态追踪的生产环境

**所需参数**：

| 参数                       | 说明                           |
| -------------------------- | ------------------------------ |
| `ALI_ACCESS_KEY_ID`      | 阿里云 AccessKey               |
| `ALI_ACCESS_KEY_SECRET`  | 阿里云 SecretKey               |
| `ALI_TTS_CODE`           | TTS 模板编号（审核通过后获得） |
| `ALI_CALLED_SHOW_NUMBER` | 主叫号码（公共模式可不填）     |
| `ALI_CONTACT_GROUP_NAME` | 联系人组名称（可选）           |

### 方案 4：腾讯云 VMS

**API**：语音通知 `SendTtsVoice`

**当前状态**：未开发

**优势**：

- 接听确认：`voice_callback` 推送 `result` 字段（`0`=接通 `1`=未接 `2`=异常）
- 回调简单：一个 `result` 字段判断一切
- 支持 HTTP 回调 + 主动拉取两种模式
- 未接通不收费

**劣势**：

- 需要企业认证
- 需要语音模板审核
- 未写代码，需从零开发

**适合场景**：已经在腾讯云生态内的企业

**所需参数**：

| 参数           | 说明             |
| -------------- | ---------------- |
| `SecretId`   | 腾讯云 SecretId  |
| `SecretKey`  | 腾讯云 SecretKey |
| `SdkAppId`   | 语音应用 ID      |
| `TemplateId` | 语音模板 ID      |

---

### 方案 5：华为云 VoiceCall

**API**：`POST /createVoiceNotification`

**当前状态**：未开发

**优势**：

- 接听确认：`statusUrl` 回调含通话状态和时长
- 支持 TTS 文本转语音（模板变量）
- 收号功能（可收集按键输入）
- AKSK 签名认证

**劣势**：

- 需要企业认证 + TTS 模板审核
- API 较复杂（Base64 编码回调 URL 等）
- 未写代码，需从零开发

**适合场景**：已经在华为云生态内的企业

**所需参数**：

| 参数           | 说明             |
| -------------- | ---------------- |
| AK             | 华为云 AccessKey |
| SK             | 华为云 SecretKey |
| `displayNbr` | 主叫号码         |
| `templateId` | TTS 模板 ID      |

---

### 方案 6：钉钉

**状态**：❌ 不可用

钉钉开放平台提供 `/topapi/call/calluser` 接口，但「智能办公电话」自 2025 年起审核系统升级，**新企业无法开通**。已开通企业可继续使用。

**结论**：不适合新接入。

---

### 方案 7：飞书

**状态**：❌ 不可用

飞书仅有「电话加急」功能（对已发送消息加电话通知）和「PSTN 入会」（电话拨入会议），**无独立 PSTN 外呼 API**。

**结论**：无法满足需求。
