# 堡垒机定时电话告警实现方案

## 一、架构总览

```
堡垒机 shell 循环脚本（每60秒轮询）
        │
        判断当前时间 ∈ [18:00, 08:00)
        判断分钟数满足触发条件
        │
        符合 → docker exec -i nightingale sh -c "python3 notify.py"
        │
notify.py（容器内）
  ├─ ① 解析 stdin JSON → 转换为内部格式
  ├─ ② collect_phones → 提取手机号
  ├─ ③ resolve_ctyun_group_names → 确定天翼云联系人组
  ├─ ④ sync_n9e_phones_to_ctyun_groups → 手机号同步到天翼云组
  ├─ ⑤ query-contact-groups → get-token → send-message
  └─ ⑥ 天翼云 API 拨出电话
        │
    电话振铃
```

## 二、两个调度脚本

| 脚本路径 | 频率 | 组名 | 手机号 | 日志 |
|---------|------|------|--------|------|
| `~/Nightingale/devops/night-call-15min.sh` | 每 15 分钟 | `NightCall-15min` | 15921882531 | `/root/n9e-night-call-15min.log` |
| `~/Nightingale/devops/night-call-30min.sh` | 整点 / 半点 | `NightCall-30min` | 17621743779 | `/root/n9e-night-call-30min.log` |

### 15 分钟脚本

```bash
#!/bin/bash
LOG=/root/n9e-night-call-15min.log
PHONES='["15921882531"]'

while true; do
    HOUR=$(date +%H)
    MIN=$(date +%M)
    if [ "$HOUR" -ge 18 ] || [ "$HOUR" -lt 8 ]; then
        if [ $((MIN % 15)) -eq 0 ]; then
            echo "$(date) 15分组触发" >> $LOG
            echo '{"sendtos":["15921882531"],"events":[{"RuleName":"夜班15分钟","Severity":1}],"tpl":{"content":"【夜莺夜班通知】请确认清醒状态。"}}' \
              | docker exec -i -e CTYUN_MONITOR_GROUP_NAMES=NightCall-15min nightingale \
                sh -c "python3 /app/etc/script/notify.py" >> $LOG 2>&1
            sleep 90
        fi
    fi
    sleep 60
done
```

### 30 分钟脚本

```bash
#!/bin/bash
LOG=/root/n9e-night-call-30min.log
PHONES='["17621743779"]'

while true; do
    HOUR=$(date +%H)
    MIN=$(date +%M)
    if [ "$HOUR" -ge 18 ] || [ "$HOUR" -lt 8 ]; then
        if [ "$MIN" -eq 0 ] || [ "$MIN" -eq 30 ]; then
            echo "$(date) 30分组触发" >> $LOG
            echo '{"sendtos":["17621743779"],"events":[{"RuleName":"夜班30分钟","Severity":1}],"tpl":{"content":"【夜莺夜班通知】请确认清醒状态。"}}' \
              | docker exec -i -e CTYUN_MONITOR_GROUP_NAMES=NightCall-30min nightingale \
                sh -c "python3 /app/etc/script/notify.py" >> $LOG 2>&1
            sleep 90
        fi
    fi
    sleep 60
done
```

## 三、notify.py 传给天翼云 API 的数据

### stdin JSON（V8 Script 通知格式）

```json
{
  "sendtos": ["15921882531"],
  "events": [
    {
      "RuleName": "夜班15分钟",
      "Severity": 1
    }
  ],
  "tpl": {
    "content": "【夜莺夜班通知】请确认清醒状态。"
  }
}
```

### 天翼云 API 三步调用

**① 查联系人组 → 获取 contactGroupID**

```
POST https://monitor-global.ctapi.ctyun.cn/v4.1/monitor/query-contact-groups
Body: {"name": "NightCall-15min", "pageNo": 1, "pageSize": 50}
返回: contactGroupID
```

**② contactGroupID → 获取 token**

```
GET https://monitor-global.ctapi.ctyun.cn/v4/monitor/get-token?contactGroupID=xxx
返回: token
```

**③ 发送语音**

```
POST https://monitor-global.ctapi.ctyun.cn/v4/monitor/send-message
Body: {"method": ["voice"], "token": "xxx", "content": "1"}
返回: {"statusCode": 800} → 成功
```

### 鉴权方式

- AK/SK：`.env` 中 `CTYUN_AK` / `CTYUN_SK`
- 签名算法：HMAC-SHA256 `Eop-Authorization`
- notify.py 每次请求动态计算签名

### 关键配置（.env）

| 变量 | 值 | 用途 |
|------|-----|------|
| `CALL_PROVIDER` | `ctyun-monitor` | 电话提供商标识 |
| `CTYUN_AK` | `1a8612e4...` | 天翼云 AccessKey |
| `CTYUN_SK` | `8e2f35f5...` | 天翼云 SecretKey |
| `CTYUN_METHODS` | `voice` | 语音通知 |
| `CTYUN_MONITOR_ENDPOINT` | `monitor-global.ctapi.ctyun.cn` | 天翼云 API 域名 |

### DNS

```yaml
# docker-compose.yml nightingale 服务下
dns:
  - 114.114.114.114
  - 8.8.8.8
```

堡垒机宿主机 `/etc/resolv.conf` 为空，容器 DNS 手动指定。

## 六、时间调度逻辑

```
夜班时段: 18:00 ~ 次日 08:00

    15分钟脚本          30分钟脚本
    ──────────          ──────────
    18:00  ✅           18:00  ✅
    18:15  ✅           18:30  ✅
    18:30  ✅           19:00  ✅
    18:45  ✅           19:30  ✅
    19:00  ✅           20:00  ✅
    ...                 ...
    07:45  ✅           07:30  ✅
    08:00  ❌ 结束      08:00  ❌ 结束
```

白天 08:00~18:00 不触发。
