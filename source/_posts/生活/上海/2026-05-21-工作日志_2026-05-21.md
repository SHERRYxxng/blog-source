## 二、二类节点修复

### 2.1 表格无数据

**根因**：`CTYunLevel2EcsDetails` 缺少 `instanceID` 字段，前端 `el-table` 的 `:row-key="row=>row.instanceID"` 全部为 `undefined`，导致表格无法渲染数据行。

**修复**：实体类增加 `getInstanceID()` 返回 `ecsId`
- 文件：`CTYunLevel2EcsDetails.java`

### 2.2 Tab 切换无效

**根因**：`@tab-click` 事件绑定异常，点击「二类节点」时不触发 `handleTabClick`

**修复**：改用 `watch: { activeTab() { ... } }` 监听 `v-model` 变化自动加载数据，移除 `@tab-click` 依赖
- 文件：`cloud-host.vue`

### 2.3 重复数据累积

**根因**：`syncEcs()` 每次调用 `ecsMapper.insert(resultList)` 只追加不删除旧数据，定时任务每小时跑一次导致 178 台累积为 3560 行

**修复**：插入前加 `ecsMapper.delete(null)` 先清空再写入
- 文件：`CTYunLevel2EcsServiceImpl.java`

### 2.4 API 并发冲突

**根因**：`getEcsInstanceDetail()` 多次并发调用导致数据互相覆盖

**修复**：添加 `_detailLoading` 防并发锁

## 四、一类节点修复

### 4.1 数据不完整

**根因**：查询按 `batchId` 过滤，只显示最新批次（16台），隐藏了其余 126 个批次（3781台）

**修复**：去掉 `batchId` 查询条件，改为直接用 `instance.id IN (SELECT MAX(id) ... GROUP BY instance_id)` 去重
- 文件：`CTYunEcsInstanceDetailServiceImpl.java`、`CTYunEcsInstanceDetailMapper.xml`

### 4.2 缺少账号来源

**根因**：数据表已有 `account_id` 但查询未关联账号名称

**修复**：
- SQL 添加 `LEFT JOIN cmdb_asset_account aa ON instance.account_id = aa.id` + `aa.alias AS accountAlias`
- VO 添加 `accountAlias` 字段
- 字典添加「来源账号」列定义
- 修复 `convertEcsInstanceDetailsEntityToJsonMap` 漏映射问题
